Apr 1, 2019 | Tutorial

How to use Github - mergetool

This is how github works:

Suppose two of you are working on a same branch, then if your partner modify a file and push it, and you haven`t make any modification on the same file, that your local file will be overide after you pull it.

However, if you two have made modification on the same file, then you have to merge it manually.

Originally, github will show the difference by:

Where between “<<<<<” and “=====” are your local commit, and
between “=====” and “>>>>>” are the remote difference.

However, it is wise to use a handy mergetool as it`s easy to use and shows the difference explicitly, like this:

where the final file is in the middle, the local is on the left and remote on the right.

To use mergetool, input the following in your terminal:

Then, when git tells you that current branch need merging, do

git mergetool

Sometimes you don`t want to do fast-forward(the operation that mention in how github works), you can ask the other contributor to push it to a new branch and fetch it, and then do

git difftool another-branch . //this line will show you all the difference between another-branch and the local directory

You can also see all the difference of current and previous commits by

git difftool HEAD^ HEAD
